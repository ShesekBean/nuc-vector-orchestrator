# Runtime Boundaries

## Vector Runtime Security Rules

Vector is a thin gRPC client only. It runs OSKR Linux but does NOT:
- Clone the repository
- Access GitHub
- Run inference or ML models
- Run Docker containers
- Run Python scripts (beyond wire-pod client)

All compute, inference, and orchestration happens on the NUC.

## Network Boundaries

- Vector ↔ NUC: gRPC over WiFi (authenticated via wire-pod)
- NUC OpenClaw: DNS allowlist via openclaw-dns container
- Signal: JSON-RPC via openclaw-gateway container (port 18889)
- wire-pod: runs on NUC, Vector connects to it for cloud services
- NUC Intercom: HTTP on NUC port 8095 (intercom-server.py → Signal DM)

## Workspace Model

Same as nuc-orchestrator:

```
workspace/
  skills/      # Runtime skill definitions
  memory/      # Runtime memory data
  proposals/   # Suggestions generated by OpenClaw
```

OpenClaw must NOT modify repository code directly.
